¿Para qué sirve?

Para versionar código.

Software que permite registrar todos los cambios de un proyecto.

Control de versiones distribuido

Cada desarrollador tiene una copia idéntica del proyecto e historial de versiones en su equipo. Sin conexión.

Características

  • Distribuido.
  • Ramas (branch) y Fusiones (merge).
  • Integridad de Datos.

¿De dónde descargarlo?

Lista de comandos

  1. Abrir GIT BASH
    • Dirigirse a la carpeta contenedora
    • Hacer click derecho
    • Seleccionar "Git Bash Here"
  2. Configuración inicial por única vez (los datos van a quedar guardados en la computadora)
    • git config --global user.name "Tu Nombre"
    • git config --global user.email tuemail@gmail.com
    • git config --list (para corroborar que los datos fueron cargados correctamente)
  3. Para empezar a trabajar con git en nuestro proyecto
    • git init
  4. Para clonar un repositorio (Proyecto) ya existente
    • git clone url
  5. Para ver en qué estado se encuentran nuestros archivos
    • git status
  6. Para agregar todos los archivos que contenga la carpeta utilizamos la primera opción. Para agregar alguno en particular podes usar la segunda cambiando el punto por el nombre del archivo (Ejemplo "index.html").
    • git add .
    • git add index.html
  7. Versionamos nuestros archivos dando un mensaje descriptivo
    • git commit -m "Mensaje"
  8. Códigos de commits realizados
    • git log --oneline
  9. Crear una nueva rama colocando el nombre que queremos (Ejemplo "nueva_rama")
    • git branch nombre_rama
  10. Mostrar ramas
    • git branch -l
  11. Eliminar rama
    • git branch -D nombre_rama
  12. Para moverse a una rama utilizamos la primer opción. La segunda es para volver a una versión anterior indicando el código que obtenemos con el paso n° 7.
    • git checkout nombre_rama
    • git checkout código
  13. Para fusionar los cambios de las distintas ramas debemos estar en MASTER.
    • git merge nombre_rama
